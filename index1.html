Blog wireframes

Blockframes

My first step to the creation of my blog was to design some potential blockframes in order to just roughly see how my website would look interms of where divs would go and how i could consider putting the elements in certain positions on the page. The benefit to doing this step before any other wireframing is to understand and visualize how content would be positioned so it can be easier to create low-fidelity wireframes which is next.




Low-fidelity Wireframes

The next step in my design process was low-fidelity wireframes which i added actual header and div names with the added annotations so that i could more contextually visualize what content would go in which divs where the were positioned which is what i had already completed and planned in the blockframing process.




High-fidelity Wireframes

The last step of my wireframing process was to complete some high-fidelity wireframes which is the next step after low-fidelity as this step is to add my proposed content to th correct positions that i planned in the previous wireframes. This is where i could completely visualize how my website is going to look before the next step which is actually to start coding. I had made a couple of minor changes to the layout in my high-fidelity wireframes which were inclusive of my Navbar and the sizing of my elements as doing these specific wireframes made me realize that elements need to adapt for presentability when the actual elements are added.




Techniques used in Blog Design

  • Nav bar - Simple layout and kept in top right of header div (to implement Jakob’s law) making it easier for users to navigate themselves around the website as most websites put the navbar here. I also added a background color and a text color to the navbar on hover so that it gives the users an extra obvious signal about which link they are clicking and when the link has actually been clicked.

  • Images/video - Adding a “title text” attribute in order to make images and video accessible to even users with disabilities (considering ethics) (users with vision disabilities when using screen readers will enable all the images/videos to be read out aloud to the user by the computer as to what the are.

  • Balance - Tried keeping a balance between text and image on each page so users wont get bored reading too much of one or the either.

  • Alignment - Aligned the elements in each section with each other whilst also aligning the header and footer elements with each other in order to maintain a high presentability. Aligned text on every page to the left to keep text layout consistent throughout each page and help it complement the centre align of the image and subheader whilst also fitting presentably between the section div.

  • Proximity - Kept all elements on each page relatively close to each other but at a suffiecient enough gap so they can be distinctive but still relate to each other on the page so users can make the connections within elements and their details whilst in the process, not getting confused reading cluttered information (Implementing the law of proximity)

  • Contrast - Chose a color scheme of orange grey and white for the backgrounds of the divs which i chose based on the fact that they contrast well with each other and also help compliment and make the black colour of the text stand out more to readers enhancing readability. Through this process i was also trying to implement the aesthetic usability effect which would make users be more attracted to my website and use it more than other potential blogs. Didnt choose too many colors to overpresent the website and turn the attention away from the information.

  • Elements - Tried not to add too many elements on each page which may cause users to be stressed/or confused as to where or what to read (implementing Hick’s Law) as i wanted to make the reading process an easy flow of movement for users.

  • Images - I took all photos into photoshop and reduced the resolution in order to decrease the file size of the image (Implementing the law of Doherty Threshold) so the website would load fast, making users stay on it without frustrations.

  • Images - Also add opacity on hover so users can know that the image is able to be clicked to see an enlarged version of it.

  • Slideshow - Along with the clickability of the images, i also added a slideshow at the bottom of the homepage in order to make my website more engaging and interesting through the use of interactive elements like the controlability of the videos as well in order to build a better relationship between the user and the website which in turn will cause them to use my website for larger periods of time.
  • Repition - Repeated layouts, fonts, and color throughout each page in order to keep reading processes the same for users and keeping the consistency of the originality of theme of my website.

  • Typography - For all the text in my website, I used the Open Sans font as through research, i found that one of the best fonts to use for a blog type of website is Open Sans, and i kept this consistent throughout each page to maintain the theme.

Javascript Baby Steps Tutorial

This tutorial was all about learning basic Javascripting skills which involved learning how to make a button function to make a certain type of motion picture. Basically the process involved using a gif image which i got from google and taking screenshots of the starting motion and then another one of the finishing motion which you would then create separate functions for each screenshot within the same function system and connecting a button functionality to the system so on clicking the button, the screenshots change in a system of on and off.
